--------------------------------------------------------------------------------
IMPORTANT:
--------------------------------------------------------------------------------

❌ DON'T USE:  python main.py <command>
✅ ALWAYS USE:  run.bat <command>

WHY?
- run.bat activates the virtual environment (venv\)
- venv\ contains all the installed packages
- python main.py uses global Python (packages not there)

--------------------------------------------------------------------------------
VIRTUAL ENVIRONMENT EXPLAINED:
--------------------------------------------------------------------------------

Think of venv\ like node_modules/ for npm:

Node.js              →  Python
--------------------------------------
npm install          →  setup.bat
node_modules/        →  venv/
npm run dev          →  run.bat command
package.json         →  requirements.txt

Both provide:
- Isolated dependencies
- No conflicts between projects
- Easy to recreate

--------------------------------------------------------------------------------
TROUBLESHOOTING:
--------------------------------------------------------------------------------

"Python is not recognized"
→ Install Python from https://www.python.org/
→ Check "Add Python to PATH" during installation
→ Restart terminal

"No module named 'rich'"
→ You're using python main.py instead of run.bat
→ Solution: run.bat status

"Virtual environment not found"
→ Run setup.bat first

Need to start fresh?
→ rmdir /s /q venv
→ setup.bat
